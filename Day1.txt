Day 1 - Git & GitHub Basics

# What is Git?

Git is a Distributed Version Control System (DVCS).
It is used to track changes in files and source code.

# What is GitHub?

GitHub is a cloud-based platform used to store and manage Git repositories online.

# Commands Learned

1. git init

Purpose:
Initialize a new Git repository.

Command:
git init

2. git status

Purpose:
Shows the current status of files in the repository.

Command:
git status

3. git add .

Purpose:
Adds all files to the staging area.

Command:
git add .

4. git commit -m "message"

Purpose:
Creates a snapshot of staged changes.

Command:
git commit -m "Added Day1 Git Notes"

5. git remote add origin <repository-url>

Purpose:
Connects local repository to GitHub repository.

Example:
git remote add origin https://github.com/Harshal-gite/harshu-gite.git

6. git branch -M main

Purpose:
Renames the current branch to main.

Command:
git branch -M main

7. git push -u origin main

Purpose:
Uploads local commits to GitHub.

Command:
git push -u origin main

# Git Workflow

Create File
↓
git status
↓

↓
git commit -m "message"
↓
git push
↓
GitHub
